## Overview

Complete re-architecture of the SD Muhammadiyah 3 school management system from a monolithic Laravel 10 Blade application (v1) to a decoupled headless architecture — Next.js 14 as the frontend and Laravel 10 as the REST API backend — deployed on Cloudflare Pages with global edge network distribution.

The decision to re-architect was driven by performance ceilings in v1 (3–5 second page load times), limited SEO capability of server-rendered Blade templates, and scalability constraints of the monolithic structure. V2 addresses all three while preserving the full feature set of v1 and adding new capabilities.

## Key Technical Features

**Headless Architecture**
The frontend and backend are fully decoupled. Next.js 14 with the App Router handles all rendering concerns: SSR for dynamic pages (news, parent portal), SSG for static content (about, facilities, achievements), and ISR for frequently updated content (schedules, staff profiles). The Laravel 10 backend exposes a versioned REST API consumed by the frontend via SWR for client-side caching and revalidation.

**Authentication and Authorization**
Laravel Sanctum for API token management. JWT stored in HTTP-only cookies (HttpOnly, Secure, SameSite) for the parent portal authentication flow. Spatie Laravel Permission manages granular RBAC for 5+ roles across both admin dashboard and parent portal. All API endpoints protected with rate limiting and middleware-level authorization checks.

**Automated Billing and Notifications**
Laravel Scheduler generates monthly invoices automatically: creates unique Virtual Accounts per student, sends QR-code payment links via Midtrans Snap, and dispatches WhatsApp notifications via Meta WhatsApp Business API. Failed notifications are queued via Laravel Queue for retry. The system maintains approximately 100% invoice generation accuracy with idempotency guards preventing duplicate invoices.

**Performance Architecture**
Edge caching via Cloudflare CDN (200+ locations). Image optimization via Next.js built-in optimizer (WebP, responsive breakpoints, lazy loading). Code splitting and dynamic imports for non-critical components. API response caching with SWR. Database-level optimization via indexed queries and eager loading to eliminate N+1 patterns. Achieved 90+ Lighthouse Score across all four categories.

**Security Implementation**
Cloudflare WAF with custom ruleset, DDoS protection, and bot management. Content Security Policy headers, CORS policy configuration, security headers (X-Frame-Options, X-Content-Type-Options, Referrer-Policy). SSL/TLS enforcement with HSTS. Bcrypt password hashing, CSRF protection on all mutation endpoints, XSS prevention via output encoding. Result: Security Rating Grade A+, zero security incidents post-launch.

**SEO Architecture**
Dynamic meta tag generation per page via Next.js metadata API. Schema.org structured data (Organization, Event, BreadcrumbList). Automatic XML sitemap generation. Open Graph and Twitter Card tags for social sharing. Robots.txt configuration. These implementations produced measurable results: 4,471 impressions and 167 clicks within the first 3 months, average position 5.9.

## Technical Challenges

**Zero-downtime Migration**
Migrating from v1 to v2 without service interruption required a blue-green deployment strategy. The v1 Laravel application remained live during v2 development. Database schema changes were additive (no destructive migrations) to maintain backward compatibility until v2 was ready for cutover.

**Rendering Strategy Selection**
Choosing between SSR, SSG, and ISR per route required careful analysis of data freshness requirements versus performance. News articles and parent portal data use SSR (always fresh). Static pages (about, facilities) use SSG (built at deploy time). Teacher profiles and class schedules use ISR with a 1-hour revalidation window — balancing freshness with edge caching efficiency.

**WhatsApp Notification Reliability**
High-volume WhatsApp notifications during billing cycles required a queued architecture. Notifications are dispatched via Laravel Queue with exponential backoff retry logic, preventing API rate limit violations and ensuring reliable delivery even under load. Achieved 98% delivery rate in production.
